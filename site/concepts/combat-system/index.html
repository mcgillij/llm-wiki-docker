    Combat System — Weapons, Targeting & Damage¶

    Overview¶

    The combat system handles all ship-to-ship and ship-to-object interactions in the game. It consists of three main layers: weapon firing (ShipWeapons + runtime classes), target acquisition (TargetingManager + CombatTargetingSolver), and damage application (via apply_damage on targetable entities). The system supports beam weapons, projectiles, homing missiles, probes, plasma charges, mining lasers, and limpets.

    Weapon Firing Architecture¶

    ShipWeapons Controller¶

    Located at /home/j/3d-space/scripts/weapons/ship_weapons.gd (2,777 lines), ShipWeapons is the central weapons controller attached to every ship node. It manages firing state, heat management, ammo tracking, beam visuals, and delegates actual projectile/beam logic to runtime classes.

    Key Properties¶

    Property Type Default Description
    muzzle_path NodePath "WeaponMuzzle" Path to muzzle node for beam origin
    max_range float 1200.0 Maximum weapon range
    base_damage float 5.0 Base damage per tick
    ramp_time float 1.5 Time to reach full damage output
    damage_tick_rate float 0.2 Interval between damage ticks
    collision_mask int 0x7fffffff Physics collision mask for projectiles

    Firing Pipeline¶

    set_firing(active, target) → _apply_group_fire_policy() → lock cone check → can_fire() check → _weapon_runtime.physics_process()
    1. Target resolution — _resolve_to_ship() walks parent chain to find ship-like ancestor (checks has_method("apply_damage"), "ship" group, or CharacterBody3D)
    2. Lock cone enforcement — Gimbaled weapons check if target is within lock_cone_dot; non-gimbaled weapons check muzzle orientation against target
    3. Heat/overheat gate — can_fire() returns false when overheated or ammo depleted
    4. Runtime dispatch — _weapon_runtime.physics_process(self, delta) handles per-frame weapon logic

    Group Fire Policy¶

    When a weapon group has multiple weapons with FirePolicy.SEQUENTIAL, firing cycles through weapons: 

    _active_group_weapon_index = (_active_group_weapon_index + 1) % _active_group_weapons.size()

    Weapon Runtime Classes¶

    Each weapon type has a dedicated runtime class that handles its specific behavior:

    Class File Behavior
    WeaponRuntimeLaser runtime/weapon_runtime_laser.gd Continuous beam, damage-on-tick, heat generation
    WeaponRuntimeRocket runtime/weapon_runtime_rocket.gd Projectile spawning with optional homing
    WeaponRuntimePlasma runtime/weapon_runtime_plasma.gd Chargeable plasma bolt, lead prediction
    WeaponRuntimeMining res://scripts/weapons/runtime/weapon_runtime_mining.gd Mining laser with resource collection
    WeaponRuntimeProbe res://scripts/weapons/runtime/weapon_runtime_probe.gd Probe deployment for scanning
    WeaponRuntimeLimpet res://scripts/weapons/runtime/weapon_runtime_limpet.gd Limpet drone deployment (60s lifetime)

    Each runtime class implements: - physics_process(weapons, delta) — per-frame logic - on_fire_stop(weapons, was_firing) — cleanup on fire stop - on_fire_stop_immediate(weapons) — immediate abort - allows_fire_without_target() — whether targetless firing is allowed - should_enforce_lock_cone() — lock cone enforcement flag - should_check_muzzle_cone() — muzzle orientation check flag

    Weapon Data Resource¶

    Located at /home/j/3d-space/scripts/resources/weapon_data.gd (56 lines), WeaponData is a Godot Resource that defines all weapon parameters. It's assigned to ShipData which is then applied to ships at runtime.

    Weapon Types¶

    • "laser" — Beam weapon, continuous damage
    • "rocket" — Projectile with optional homing
    • "plasma" — Chargeable projectile with lead prediction
    • "mining" — Mining laser for resource extraction
    • "probe" — Scanner probe deployment
    • "limpet" — Limpet drone (60s lifetime)

    Key Parameters¶

    Parameter Type Description
    weapon_type String Weapon category
    weapon_range float Maximum effective range
    damage float Base damage per tick
    ramp_time float Time to reach full damage
    gimbaled bool Can swivel independently of ship orientation
    gimbal_turn_rate float Degrees/second for gimbaled weapons
    lock_cone_dot float Cosine of max angle from muzzle forward (0.96 ≈ 16°)
    ammo_capacity / ammo_per_shot int Ammo system (0 = infinite)
    fire_interval float Minimum time between shots
    projectile_speed float Speed for projectile weapons
    homing_turn_rate float Turning rate for homing projectiles
    lock_time float Time required to achieve lock-on

    Heat System¶

    heat_capacity: 100.0        # Total heat units before overheat
heat_rate: 25.0             # Heat generated per second while firing
heat_recharge: 20.0         # Heat dissipated per second when idle
heat_recharge_delay: 1.0    # Delay before recharge begins after stopping fire
overheat_cooldown: 2.0      # Enforced cooldown duration when overheated
overheat_resume_threshold_pct: 25.0  # Resume firing when heat drops below this %

    Plasma Charge System¶

    Plasma weapons support a charge mechanic: - charge_time_max — Maximum charge time (default 3s) - charged_damage_multiplier — Damage multiplier at full charge (×3.0) - charged_speed_multiplier — Projectile speed reduction when charged (×0.45, slower = more accurate) - charged_scale_multiplier — Visual scale increase when charged (×2.2)

    Targeting System¶

    TargetingManager¶

    Located at /home/j/3d-space/scripts/systems/targeting_manager.gd (817 lines), this manages target acquisition for the player ship. It queries the FW_TrackableRegistry for nearby objects and maintains a sorted list of targets.

    Target Groups¶

    Group Type Purpose
    "nav/player_remote" enemy Remote-controlled player ships (NPCs)
    "nav/enemy" enemy Full AI enemy ships
    "nav/enemy_lightweight" enemy Lightweight NPC contacts
    "nav/poi" neutral Points of interest
    "nav/asteroid" neutral Asteroids (special handling)
    "nav/gate" neutral Space gates
    "nav/planet" / "nav/star" neutral Celestial bodies
    "nav/station" neutral Space stations
    "nav/nebula" neutral Nebula regions
    "nav/collectible" neutral Collectible items
    "nav/convoy_freighter" friendly Convoy freighters
    "nav/convoy_escort" friendly Convoy escorts

    Target Cycling¶

    • select_next() / select_previous() — Cycle through all targets
    • select_next_enemy() — Skip to next enemy (or non-enemy if no enemies)
    • select_previous_non_enemy() — Cycle through friendly/non-hostile targets
    • Targets are sorted by distance from player ship

    Target Refresh Logic¶

    Targets are refreshed when: 1. Registry signals a change (_on_registry_changed) 2. Player ship moves beyond resort_distance_threshold (40m default) 3. Timer fires at update_interval (0.2s default) 4. Mouse click targeting is enabled and player clicks on screen

    Lightweight NPC Targeting¶

    Lightweight NPCs have reduced targeting ranges: - lightweight_contact_max_range: 4,000m (vs 8,000m for full enemies) - selected_lightweight_retention_range: 12,000m (keep selected target even if it moves out of range)

    CombatTargetingSolver¶

    Located at /home/j/3d-space/scripts/systems/combat_targeting_solver.gd (84 lines), this provides static utility methods for combat calculations.

    Intercept Point Calculation¶

    compute_intercept_point(shooter_pos, target_pos, target_vel, projectile_speed) -> Vector3

    Solves the quadratic equation for time-to-intercept: 

    a = |target_vel|² - projectile_speed²
b = 2 × (target_pos - shooter_pos) · target_vel
c = |target_pos - shooter_pos|²
t = (-b ± √(b² - 4ac)) / 2a

    Returns target_pos directly if: - Projectile speed ≤ 0.01 (beam weapons, instant hit) - No valid positive solution for t exists

    Weapon Origin & Forward¶

    • get_weapon_origin() — Returns muzzle position if available, otherwise ship track position
    • get_weapon_forward() — Returns negative Z axis of muzzle or ship transform
    • get_display_distance_for_weapon() — Clamped to [120m, 2400m], defaults to 60% of weapon range

    Damage Application¶

    TargetableEntity Protocol¶

    All damageable entities implement the TargetableEntity protocol: 

    func apply_damage(damage: float, source: Node3D = null) -> void
func get_track_position() -> Vector3
func get_velocity() -> Vector3
func get_display_name() -> String

    The TargetingManager uses these methods to query target properties without tight coupling. Damage is applied through the apply_damage() method which is called by weapon runtime classes when a hit is confirmed (beam tracking or projectile collision).

    Beam vs Projectile Damage Models¶

    Beam weapons (lasers): Continuous damage ticked at damage_tick_rate while firing and locked on target. Damage ramps up over ramp_time. Heat is generated continuously during fire.

    Projectile weapons (rockets, plasma): Single-hit damage on collision. Homing projectiles use homing_turn_rate to steer toward target. Plasma bolts use lead prediction based on projectile speed.

    Relationships¶

    • Consumed by flight-controller for player weapon firing (via set_weapon_firing())
    • Consumed by ai-subsystem for NPC combat behavior (AiFlightController.set_fire_target() / set_firing())

    • Integrates with faction-manager through faction-based targeting rules (enemy vs friendly)
