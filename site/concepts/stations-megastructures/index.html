    Stations & Megastructures¶

    Procedural station system and large-scale megastructures that populate every world-generation system.

    Overview¶

    Every generated system contains stations (player-accessible hubs) and megastructures (visual/lore landmarks). Both are defined in SystemRecipe data created by the procedural-level-builder pipeline, then instantiated at runtime with deterministic parameters derived from the galaxy seed.

    Key files: - /scripts/resources/station_data.gd — StationData resource (345 lines) - /scripts/level/station_docking_manager.gd — Docking flow + service menu (1040 lines) - /scripts/ui/station_market_popup.gd — Buy/sell market UI (650 lines) - /scripts/resources/system_recipe.gd — MegastructureData definition (lines 392–402)

    Station Types & Services¶

    StationType Enum (StationData.StationType)¶

    Type Display Name Default Population Docking Bays Default Services
    OUTPOST Outpost 500 2 Refuel, Restock, Information
    STATION Station 5,000 4 Trade, Missions, Refuel, Repair, Restock
    MEGASTATION Mega Station 50,000 12 All services + Crew
    MILITARY Military Base 3,000 6 Missions, Refuel, Repair, Restock (security: 0.9)
    RESEARCH Research Station — — Information, Missions
    MINING Mining Hub — — Trade, Refuel, Repair
    AGRICULTURAL Agricultural Station — — Trade, Refuel, Restock
    INDUSTRIAL Industrial Complex — — Trade, Refuel, Repair
    HABITAT Habitat — — Trade, Missions, Information
    PRISON Penal Colony — — Information only
    SCIENTIFIC Science Outpost — — Information, Missions

    StationSize Enum¶

    • SMALL: < 1,000 inhabitants
    • MEDIUM: 1,000–10,000
    • LARGE: 10,000–100,000
    • MASSIVE: > 100,000

    ServiceType Enum¶

    TRADE, MISSIONS, SHIPYARD, REFUEL, REPAIR, RESTOCK, CREW, INFORMATION, BLACK_MARKET

    Each station type gets a default service set via StationData.get_default_services(). Services can be added/removed at runtime.

    Station Properties¶

    • faction_id — controlling faction (default: "independent")
    • orbit_distance_au / orbital_inclination — orbital parameters
    • tech_level — 0–10 scale
    • security_level — 0.0–1.0 (affects market restrictions, black market availability)
    • has_market — commodity trading enabled
    • market_volatility — price fluctuation factor
    • market_allowed_categories / market_restricted_categories / market_preferred_categories / market_discouraged_categories — dynamic category controls

    Docking System¶

    Detection Flow (StationDockingManager)¶

    1. Scan loop (every 0.25s): iterates all nodes in "nav/station" group, computes distance from player ship
    2. Prompt display: when nearest station ≤ docking_range (default 1200 units), shows on-screen prompt
    3. Player input: pressing flight_interact opens the docking menu

    Service Menu (ActionMenuOverlay)¶

    The menu is built dynamically in _refresh_menu_for_station() and includes:

    • Refuel — restores jump fuel, cost = 4 cr/unit (from REFUEL_COST_PER_UNIT)
    • Repair — restores hull integrity, cost = 6 cr/unit (REPAIR_COST_PER_UNIT)
    • Restock — refills ammo + probes, cost = 2 cr/ammo-unit + 5 cr/probe (AMMO_COST_PER_UNIT, PROBE_COST_PER_UNIT)
    • Open Market — launches station-market-popup
    • Accept Convoy Contract — from SystemEventRuntime.get_merchant_convoy_offer()
    • Accept Pirate Raid — from SystemEventRuntime
    • Sell Discovery Data — sells bundled unsold discoveries via TradeService
    • All Services — applies all available services at once

    Reputation Gate¶

    Docking is denied if the player's standing with the station's faction is too low. The menu shows "Docking denied by {station} ({standing})".

    Station Market (StationMarketPopup)¶

    A 650-line popup for commodity trading, opened from the docking menu or via open_at() / open().

    Layout¶

    • Sell list — player's cargo items available to sell (filtered by category)
    • Buy list — station's stock (filtered by category)
    • Category filters — dropdowns populated from TradeService.get_all_categories()
    • Quantity spinners — per-item buy/sell amounts
    • Quote labels — live price quotes with color coding
    • Transaction history — last 14 entries

    Badge System (color-coded item categories)¶

    Badge Color Meaning
    Allowed default white Accepted at this station
    Preferred green Better buy/sell rates
    Discouraged orange Weaker rates
    Restricted red Blocked (high security stations)
    Black Market orange Contraband accepted
    Demand Spike / Oversupply — Active market event

    Trade Flow¶

    1. Player selects item → quantity spinner updates max value
    2. Quote computed via TradeService.quote_sell() or quote_buy()
    3. Buy/sell button triggers async request (request_sell / request_buy)
    4. Result tracked in _pending_request_ids, resolved on transaction_completed signal
    5. UI refreshes automatically

    Security Integration¶

    Market help text shows current station security level and whether black market is available: "Contraband blocked at high security" vs "Black Market allows contraband".

    Megastructures¶

    All megastructures share a common interface pattern defined by SystemRecipe.MegastructureData:

    class MegastructureData:
    var structure_id: String = ""
    var kind: int          # MegastructureKind enum value
    var shape_kind: int    # MegastructureShapeKind enum value
    var faction_id: String = ""
    var display_name: String = ""
    var anchor_entity_ids: PackedStringArray
    var control_points: PackedVector3Array
    var metadata: Dictionary  # kind-specific parameters
    var seed_value: int = 0

    MegastructureKind Enum (20+ types)¶

    Core 13: HIGHWAY, PLANET_RING, RING_HABITAT, SOLAR_ARRAY, SHIPYARD, SENSOR_ARRAY, AQUEDUCT, HABITAT_RING, DEFENSE_PLATFORM, ORBITAL_TETHER, DYSON_SWARM, DYSON_SHELL_SEGMENT, SATELLITE_CONSTELLATION

    New additions (post-wiki): - ARCLOGY_CLUSTER — interconnected arcology clusters (ArcologyClusterMegastructure) - FACTORY_COMPLEX — industrial manufacturing complexes (FactoryComplexMegastructure) - NANO_FAB_LAB — nanofabrication laboratories (NanofabLabMegastructure) - RECYCLING_PLANT — material recycling and processing plants (RecyclingPlantMegastructure) - REFINERY_STATION — resource refining stations (RefineryStationMegastructure) - WAREHOUSE_HUB — logistics and storage hubs (WarehouseHubMegastructure) - ORBITAL_CITY — large-scale orbital cities with surface integration (OrbitalCityMegastructure) - SPACE_COLONY_HUB — deep-space colony hubs with multi-ring design (SpaceColonyHubMegastructure) - PROPULSION_TEST_FACILITY — engine testing facilities (PropulsionTestFacilityMegastructure)

    MegastructureShapeKind Enum (8 shapes)¶

    POINT, PATH, RING, ARC, CLUSTER, BAND, SHELL_SEGMENT, AREA

    Individual Megastructure Types¶

    DefensePlatform (DefensePlatformMegastructure)¶

    • Profiles: gate_bastion (radial turret arms), station_redoubt (dual-side fortifications)
    • Components: central keep, radial arms with turret pairs, perimeter lights
    • Parameters: cluster_radius, turret_count, bastion_height, platform_width
    • Visuals: hull shader (defense_platform_hull.gdshader) + accent shader with pulse animation

    HabitatRing (HabitatRingMegastructure)¶

    • Shape: full ring or arc segments (72-segment procedural mesh)
    • Components: ring band, habitation modules (MultiMesh), beacon markers
    • Parameters: inner_radius, outer_radius, arc_fraction, start_angle_degrees, module_count, beacon_count
    • Visuals: habitat_ring_band.gdshader with window glow and stripe patterns

    Shipyard (ShipyardMegastructure)¶

    • Profiles: drydock_spine (linear keel), side_arm_frame (lateral bays), circular_dock_frame (ring dock)
    • Components: main frame, guide light strips, dock bay markers
    • Parameters: frame_length, frame_width, frame_height, dock_bay_count, guide_light_count
    • Visuals: hull shader with panel grid + accent shader for lighting

    DysonSwarm (DysonSwarmMegastructure)¶

    • Shape: orbital energy collection bands around a star
    • Components: multiple concentric bands, solar panels (QuadMesh), support structures
    • Parameters: band_radius, panel_width, panel_height, arc_fraction, band_count, band_spacing
    • Visuals: dyson_swarm_cluster.gdshader with energy pulse animation across panel grid

    SolarArray (SolarArrayMegastructure)¶

    • Shape: collector panels on a central spine structure
    • Components: spine beam, panel arrays, hub connectors
    • Parameters: band_radius, panel_width, panel_height, arc_fraction, band_count, band_spacing
    • Visuals: solar_collector_panel.gdshader with scan-line effect

    Aqueduct (AqueductMegastructure) & OrbitalTether (OrbitalTetherMegastructure)¶

    • Shape: path-based ribbon structures (connect two points via Curve3D)
    • Components: extruded mesh along curve, endpoint anchors, brace markers
    • Parameters: width, height, style ("tube", "industrial_pipe", "lattice")
    • Visuals: shared megastructure_strip.gdshader (aqueduct) or orbital_tether_strip.gdshader with pulse animation

    SensorArray (SensorArrayMegastructure)¶

    • Styles: relay_dishes (parabolic dishes on masts), sensor_spires (tall antenna spires)
    • Components: central hub, element array (masts + dishes/spires), beacon ring
    • Parameters: cluster_radius, element_count, mast_height, dish_radius
    • Visuals: hull shader with panel grid + accent shader with pulse animation

    Condition System (All Megastructures)¶

    Every megastructure supports four visual conditions that affect detail density and emission:

    Condition Detail Scale Emission Multiplier
    pristine 1.0x base strength
    weathered 0.86–0.94x ~0.78–0.92x
    degraded 0.52–0.72x ~0.52–0.68x
    damaged 0.35–0.72x ~0.02–0.30x

    Condition is stored in metadata and applied during _build_visuals() to scale element counts, reduce panel density, and dim emission effects.

    Visual Construction Pattern¶

    All megastructures follow the same construction pattern: 1. configure() — sets metadata from recipe data, adds to nav groups ("nav/megastructure" + type-specific group) 2. _clear_runtime_children() — frees previous children for regeneration 3. _build_visuals() — creates procedural geometry using BoxMesh, QuadMesh, ArrayMesh, and MultiMeshInstance3D 4. Custom shaders per-type with parameters: tint, emission_strength, stripe_density/speed/strength, metallic/roughness

    Nav Groups¶

    Each megastructure adds itself to "nav/megastructure" plus a type-specific group for targeting: - "nav/defense_platform", "nav/habitat_ring", "nav/shipyard", "nav/dyson_swarm", "nav/solar_array", "nav/aqueduct", "nav/orbital_tether", "nav/sensor_array"

    Integration with World Generation¶

    Megastructures and stations are generated by the procedural-level-builder pipeline: 1. SystemGenerator creates a SystemRecipe from GalaxySystemData.system_seed 2. Recipe includes arrays of StationData and MegastructureData entries 3. At runtime, ProceduralLevelBuilder instantiates each megastructure by calling its configure() method with the recipe data 4. Stations are placed at orbital distances from the star using deterministic positioning
