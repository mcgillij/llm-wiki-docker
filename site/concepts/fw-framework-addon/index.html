    FW Framework Addon — Service Classes & Architecture¶

    Overview¶

    The FW (Framework) addon is a collection of reusable service classes located in /home/j/3d-space/addons/fw/. It provides core infrastructure for the 3dSpace project including configuration persistence, scene transitions, state machines, audio management, HTTP networking, and async save/load with schema migration. Services are configured via FW_ConfigService which acts as a central config hub — each service connects to it on initialization to read settings and listen for changes.

    Directory Structure¶

    addons/fw/
├── Bootstrap/FW_Bootstrap.gd          # Initialization orchestrator
├── Config/
│   ├── FW_ConfigService.gd            # Central config persistence (INI-style)
│   ├── FW_FrameworkDefaults.gd        # Default configuration values
│   └── FW_WindowPrefs.gd              # Window preference management
├── State/
│   ├── FW_StateMachine.gd             # Explicit state machine pattern
│   └── FW_State.gd                    # Base state class (optional)
├── Scenes/FW_SceneRouter.gd           # Scene transitions with animated effects
├── Audio/FW_AudioService.gd           # SFX pool + music management
├── Net/FW_NetService.gd               # HTTP API client with health checks
├── Save/FW_SaveService.gd             # Async save/load with schema migration
├── Resources/
│   ├── FW_ResourceCache.gd            # Resource preloading cache
│   └── FW_PreloadQueue.gd             # Deferred preload queue
├── Input/FW_InputRebindService.gd     # Input action rebinding
├── Platform/
│   ├── FW_Platform.gd                 # Platform detection (Steam Deck, etc.)
│   └── Steam/FW_SteamService.gd       # Steam platform integration
└── Scenes/LevelHelpers/level_manager.gd  # Level loading helpers

    FW_ConfigService¶

    Located at /home/j/3d-space/addons/fw/Config/FW_ConfigService.gd (89 lines), this is the central configuration service. It wraps Godot's ConfigFile class with typed getters, default value support, and change signals.

    Key Features¶

    • INI-style persistence — Saves to user://save/framework_config.cfg by default
    • Default values — Register defaults via register_defaults() before loading; fallback chain: saved value → registered default → explicit fallback
    • Typed getters — get_bool(), get_int(), get_float(), get_string(), get_vec2() with automatic type conversion
    • Change signals — Emits changed(section, key, value) on every set, enabling services to react to config changes

    API¶

    func register_defaults(defaults_by_section: Dictionary) -> void  # Register fallback defaults
func load(path: String = DEFAULT_PATH) -> void                  # Load from file (creates new ConfigFile on error)
func save() -> void                                             # Save to loaded path
func get_value(section, key, fallback=null) -> Variant          # Get with fallback chain
func set_value(section, key, value, autosave=false) -> void     # Set and optionally auto-save
func has_key(section, key) -> bool                              # Check if key exists in saved config

    Config Sections Used by Project¶

    Section Keys Purpose
    audio sfx_enabled, sfx_volume_db, music_enabled, music_volume_db Audio settings
    display rotated Screen rotation state (Steam Deck)
    net enabled, base_url, api_key, auto_disable_on_fail, healthcheck_path, healthcheck_cache_seconds Network API config

    FW_SceneRouter¶

    Located at /home/j/3d-space/addons/fw/Scenes/FW_SceneRouter.gd (507 lines), manages scene transitions with animated effects and screen rotation support.

    Scene Transition System¶

    Two transition modes: 1. Shader-based — Uses a custom shader for complex visual transitions (wipes, iris, spikes, clock sectors) 2. Lightweight fade — Simple black fade in/out for mobile devices or when no shader is configured

    Built-in Transition Presets¶

    Preset Type Description
    "Directional Wipe" 0 (directional) Single-direction wipe
    "Corner Wipe" 0 Four-corner wipe
    "Diagonal Wipe" 0 45° diagonal wipe
    "Cross-shaped Transition" 0 Expanding cross from center
    "Iris Transition" 2 (radial) Circular iris opening/closing
    "Spike Transition" 2 Star/spike pattern from center
    "Overlapping Diamonds" 2 Diamond pattern with feathering
    "Center-Clock Transition" 3 (sector) Clock-sector wipe from center
    "Corner-Clock Transition3" 3 Multi-corner clock sectors
    "Seamless Striped Flower" 3 Complex striped flower pattern

    Screen Rotation Rig¶

    For Steam Deck and portrait-mode displays: 1. Creates a Node2D rotator rig that rotates the entire viewport 90° 2. Wraps the scene in a SubViewportContainer inside the rig 3. Reparents popups/windows to the scene host viewport for correct positioning 4. Toggled via set_rotated(value) with config persistence

    API¶

    func change_scene(path: String, transition_params={}) -> void  # Load new scene with transition
func set_rotated(value: bool, persist=true) -> void           # Toggle screen rotation
func toggle_rotation(persist=true) -> void                    # Toggle and persist
func setup_rotation_rig() -> void                             # Build rotation rig manually
func teardown_rotation_rig() -> void                          # Remove rotation rig

    Signals¶

    • rotation_changed(is_rotated: bool) — Rotation state changed
    • transition_started(path: String) — Transition animation began
    • transition_finished(path: String, ok: bool) — Scene swap completed

    FW_StateMachine¶

    Located at /home/j/3d-space/addons/fw/State/FW_StateMachine.gd (77 lines), implements an explicit state machine pattern for managing behavior modes. States can be either FW_State objects or plain dictionaries of callables.

    State Lifecycle¶

    start(initial_state) → on_enter("", data)
transition_to(next, data) → on_exit(prev) → on_enter(prev, data) → emit state_changed(prev, next)
tick(delta) → on_update(delta)          # Called every frame if auto_process=true
send_event(event) → on_event(event)     # Dispatch events to current state

    State Definition Formats¶

    Dictionary format (simple): 

    state_machine.add_state("idle", {
    "on_enter": Callable(self, "_on_idle_enter"),
    "on_update": Callable(self, "_on_idle_update"),
    "on_event": Callable(self, "_on_idle_event"),
})

    Object format (full): 

    class MyState extends FW_State:
    func on_enter(prev_state, data): ...
    func on_exit(next_state): ...
    func on_update(delta): ...
    func on_event(event): ...

state_machine.add_state("combat", my_state_instance)

    API¶

    func add_state(name: StringName, state: Variant) -> void     # Register a state
func has_state(name: StringName) -> bool                     # Check if state exists
func get_state() -> StringName                               # Get current state name
func start(initial: StringName, data=null) -> Dictionary     # Initialize and enter initial state
func transition_to(next: StringName, data=null) -> Dictionary  # Transition to new state
func tick(delta: float) -> void                              # Process current state update
func send_event(event: Variant) -> void                      # Dispatch event to current state

    FW_AudioService¶

    Located at /home/j/3d-space/addons/fw/Audio/FW_AudioService.gd (154 lines), manages sound effects and music with a reusable SFX player pool.

    Architecture¶

    • SFX Pool — Pre-creates sfx_pool_size (default 8) AudioStreamPlayer instances on the SFX bus. Reuses idle players for new sounds.
    • Music Player — Single dedicated AudioStreamPlayer on the Music bus.
    • Volume Control — Reads volume settings from FW_ConfigService (audio.sfx_volume_db, audio.music_volume_db). Mutes to -80dB when disabled.

    API¶

    func register_sfx(id: StringName, stream: AudioStream) -> void    # Register SFX by ID
func register_music(id: StringName, stream: AudioStream) -> void  # Register music by ID
func play_sfx(id, volume_db_offset=0.0, pitch_scale=1.0) -> void  # Play SFX (pooled)
func play_music(id, volume_db_offset=0.0) -> void                 # Play/switch music
func stop_music() -> void                                         # Stop current music
func bind_button_group(group_name="all_buttons", sound_id="ui.click") -> void  # Auto-play SFX on button press

    FW_NetService¶

    Located at /home/j/3d-space/addons/fw/Net/FW_NetService.gd (257 lines), provides HTTP API access with health checking and automatic disable-on-failure.

    Health Check System¶

    • Periodic health checks to base_url + healthcheck_path (default "/")
    • Results cached for healthcheck_cache_seconds (default 2s) to avoid redundant requests
    • auto_disable_on_fail flag: if true, permanently disables network after a failed request and emits network_auto_disabled signal

    Request Flow¶

    func request_json(method, path_or_url, payload, callback, extra_headers=[], scene=null, auto_disable_override=null) -> void
    1. Validates that network is enabled and base_url is set 2. Constructs full URL (joins base_url with relative path) 3. Serializes payload to JSON, builds headers (Content-Type + API key if configured) 4. Creates HTTPRequest node, attaches to scene tree 5. On completion: parses JSON response, calls callback with {ok: true/false, data/error}

    API Key Support¶

    If api_key is configured in the net section, it's automatically added as X-API-Key header to all requests.

    FW_SaveService¶

    Located at /home/j/3d-space/addons/fw/Save/FW_SaveService.gd (710+ lines), provides async save/load with schema migration and corruption recovery. This is the most complex service in the framework.

    Save Slot System¶

    • 3 slots (0, 1, 2) stored as user://save/slot_0.json, slot_1.json, slot_2.json
    • Each save file contains: schema version, metadata (created_at_unix, saved_at_unix), galaxy state, and game data

    Schema Migration¶

    The service supports automatic migration between schema versions: 

    Version -1 (legacy) → Version 0 → Version 1 (current)
    - On load: detects schema version, migrates if needed, writes back canonical format - On save: always writes current schema version (v1) - Corrupt files are backed up to user://save/corrupt/ before being replaced

    Async Save System¶

    Uses a dedicated worker thread with mutex/semaphore synchronization: 1. save_slot_async() queues the job, coalescing duplicate requests for the same slot 2. Worker thread processes jobs from queue, writes files synchronously 3. Results are collected in _async_completed array and dispatched on main thread via _process()

    Save File Format (v1 Canonical)¶

    {
  "_fw_schema_version": 1,
  "created_at_unix": 1234567890,
  "saved_at_unix": 1234567890,
  "meta": { ... },
  "data": { ... },
  "galaxy_state": { ... }
}

    Corruption Handling¶

    • Empty files → treated as corruption
    • JSON parse failures → backed up to corrupt/ directory
    • Future schema versions → rejected with error (no downgrade)
    • Migration failures → backed up and rejected

    API¶

    func load_slot(slot_id: int) -> Dictionary                              # Load save slot (synchronous, emits signal)
func save_slot(slot_id, data, galaxy_state={}, meta_overrides={}) -> Dictionary  # Save slot (sync)
func save_slot_async(slot_id, data, galaxy_state={}, meta_overrides={}) -> Dictionary  # Queue async save
func flush_async_queue(timeout_ms=3000) -> bool                         # Wait for all pending saves
func delete_slot(slot_id: int) -> Dictionary                            # Delete save slot
func get_save_metadata(slot_id: int) -> Dictionary                      # Get metadata without loading full data
func slot_exists(slot_id: int) -> bool                                  # Check if slot has a file

    Signals¶

    • save_completed(slot_id, ok, result) — Save finished (sync or async)
    • load_completed(slot_id, ok, result) — Load finished
    • slot_deleted(slot_id, ok, result) — Delete finished

    FW_Bootstrap¶

    Located at /home/j/3d-space/addons/fw/Bootstrap/FW_Bootstrap.gd, orchestrates the initialization of all framework services in the correct order:

    1. Create and configure FW_ConfigService (load defaults + save file)
    2. Configure all other services with the config service reference
    3. Set up scene router, audio, networking, and save services
    4. Ensure proper cleanup on exit (flush async saves, close threads)

    Key Files Reference¶

    File Purpose Lines
    addons/fw/Config/FW_ConfigService.gd Central config persistence with defaults and typed getters 89
    addons/fw/Scenes/FW_SceneRouter.gd Scene transitions (shader-based + fade), screen rotation rig 507
    addons/fw/State/FW_StateMachine.gd Explicit state machine with on_enter/on_exit/on_update/on_event lifecycle 77
    addons/fw/Audio/FW_AudioService.gd SFX pool (8 players) + music player, volume control from config 154
    addons/fw/Net/FW_NetService.gd HTTP API client with health checks, JSON parsing, auto-disable on failure 257
    addons/fw/Save/FW_SaveService.gd Async save/load (3 slots), schema migration (-1→0→1), corruption recovery 710+
    addons/fw/Bootstrap/FW_Bootstrap.gd Service initialization orchestrator —

    Relationships¶

    • FW_ConfigService is the foundation — all other services depend on it for configuration
    • FW_SceneRouter used by main game flow for scene transitions (title → galaxy → space)
    • FW_StateMachine consumed by ai-subsystem for NPC behavior state management
    • FW_AudioService integrated with UI via bind_button_group() for consistent sound feedback
    • FW_NetService provides API access for online features (leaderboards, cloud saves)
    • FW_SaveService handles all game persistence including galaxy state and faction data
