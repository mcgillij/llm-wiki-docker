    UI & HUD Subsystems — HUD Modules, Galaxy Map & Menus¶

    Overview¶

    The HUD system is a modular, lazy-loaded CanvasLayer that provides flight information, targeting feedback, economy data, and game state awareness. The architecture was designed to solve the problem of a monolithic 1881-line HudOverlay script blocking startup — widgets are now loaded across frames using a priority-queued bootstrap system so combat-critical elements appear immediately while expensive widgets load progressively.

    HudOverlay — Main HUD CanvasLayer¶

    Located at /home/j/3d-space/scripts/ui/hud_overlay.gd (1256+ lines), HudOverlay is the root CanvasLayer for all in-game UI. It extends Godot's CanvasLayer and registers itself in groups "ui/hud_overlay" and "hud/runtime_refreshable".

    Architecture¶

    The HUD follows a coordinator pattern: HudOverlay manages layout containers, service connections, and widget lifecycle, but individual widgets are instantiated via factory methods and loaded through the priority bootstrap system.

    Scene Structure (Layout Containers)¶

    Container Type Purpose
    %HudPanel PanelContainer Main HUD panel wrapper
    %WidgetRow HBoxContainer Navigation widgets row (navball, radar, assist vector)
    %WeaponWidgetBox VBoxContainer Weapon/countermeasure display area
    %ThrottleBoostVbox VBoxContainer Throttle and boost indicators
    %ShipInfoContainer VBoxContainer Ship info panel content
    %ShipInformation PanelContainer Ship info panel wrapper (toggleable)
    %CargoPanel / %CargoInfo PanelContainer/VBoxContainer Cargo display area
    %BoostInfo VBoxContainer Boost/travel mode indicators
    %WeaponInfo VBoxContainer Weapon status (overheat, ammo, plasma charge)
    %ShieldInfo / %HullInfo / %FuelInfo VBoxContainer Defense/status bars
    %MiningInfo VBoxContainer Mining/limpet display area
    %CreditsLabel / %ResourcesLabel Label Economy strip (always visible)
    %EconomyDetailsLabel Label Expanded economy details (toggleable)
    %IntelDock Control Utility buttons (legend, journal, cargo manifest, jobs)
    %DiscoveryLegend / %LegendLabel PanelContainer/Label Discovery info display
    %FpsAnchor / %FpsLabel Control/Label FPS counter

    All visibility is controlled via @export bools on HudOverlay, defaulting to true except debug panel:

@export var show_assist_vector: bool = true
@export var show_target_indicator: bool = true

@export var show_countermeasures: bool = true
@export var show_debug_panel: bool = false

    HudContext Enum¶

    The HUD supports three contexts that control which widgets are visible: 

    enum HudContext {
    CONTEXT_FLIGHT,   # Navball, radar, weapons, throttle, jump indicator, targeting reticle
    CONTEXT_DOCKED,   # Cargo manifest, economy strip, intel dock, jobs board
    CONTEXT_MENU,     # Minimal shell only
}
var _current_context: HudContext = HudContext.CONTEXT_FLIGHT

    Service Modules (Child Nodes)¶

    Three service modules are instantiated as child nodes during _ready(): 1. HudEconomyModule — Credits display, resource ledger, economy strip toggle 2. HudFactionModule — Faction standing notifications and UI updates 3. HudJobsModule — Discovery journal, cargo manifest modal, jobs board, discovery notifications

    Key Methods¶

    func _ready() -> void                          # Setup containers, connect services, start bootstrap on next frame
func _bootstrap_dynamic_widgets() -> void      # Create HudWidgetBootstrap and register all widgets
func _register_widget_bootstraps() -> void     # Register 24+ widgets with priorities (CRITICAL/HIGH/MEDIUM/LOW)
func _unhandled_input(event: InputEvent) -> void  # Handle HUD input actions (hail, journal, cargo manifest, jobs)

    Service Connections¶

    During _ready(), HudOverlay connects to: - TrackableRegistry — For trackable entity updates - TargetingManager — For target change events (target_changed signal) - CreditsManager (via EconomyModule) — For credit changes - FactionManager (via FactionModule) — For faction standing updates - SystemEventRuntime (via JobsModule) — For system event notifications

    HudWidgetBootstrap — Priority Queue System¶

    Located at /home/j/3d-space/scripts/ui/hud_widget_bootstrap.gd (147 lines), this is the core lazy-loading engine that replaces the old sequential widget instantiation chain. It loads widgets across multiple frames within a per-frame time budget, preventing HUD startup from blocking player control.

    Priority Levels¶

    Level Value Widgets Included Load Time Target
    CRITICAL 0 Targeting reticle, weapon groups, countermeasures, jump indicator, throttle boost, ship info, cargo, boost, travel mode, laser overheat, weapon ammo, plasma charge, limpet, mining, shield, hull, fuel First 2-3 frames (~5ms)
    HIGH 1 Navball, radar X4, assist vector, target indicator, target info, position Next 5-10 frames
    MEDIUM 2 Target view (SubViewport+Camera), navigation widget, debug panel Background / on-demand
    LOW 3 Discovery notification, active jobs, convoy waypoint, multiplayer debug As needed

    API¶

    func register(id: String, priority: int, factory: Callable, visible_property: String = "") -> void
# Register a widget for lazy bootstrap. id is unique; factory returns the instantiated Node.

func start() -> void
# Sort pending queue by priority and begin frame-by-frame loading.

func process_next() -> bool
# Process next batch within time budget (default 5ms/frame). Returns true when all loaded.

func remove_widget(id: String) -> void
# Free a widget instance and remove from queue if not yet loaded.

func toggle_widget(id: String, enabled: bool) -> void
# Runtime enable/disable: pushes to front of pending queue on enable, frees instance on disable.

func is_loaded(id: String) -> bool
func get_instance(id: String) -> Node
func has_pending() -> bool
func is_running() -> bool

    Bootstrap Flow¶

    1. _bootstrap_dynamic_widgets() creates HudWidgetBootstrap and calls _register_widget_bootstraps()
    2. Each widget is registered with a factory callable (from HudWidgetFactories) and an optional visibility property name
    3. start() sorts entries by priority into the pending queue
    4. On each frame, process_next() loads widgets until the time budget is exhausted
    5. Loaded widgets are added to _widgets array and synced with ship/trackables

    const DEFAULT_NAVBALL_SCENE := preload("res://scenes/ui/navball_control.tscn")
# ... 25+ more default scenes loaded at compile time

    Service Modules¶

    HudEconomyModule¶

    Located at /home/j/3d-space/scripts/ui/hud_economy_module.gd (237 lines), manages credits display and resource ledger.

    Credits Display: Connects to CreditsManager.credits_changed signal, formats with comma separators via _format_credits().

    Resource Ledger: Reads from ResourceLedgerManager.get_or_create_ledger() + GalaxyNavigator.get_system_coords() to show current system resources. Displays top 2 resources on the main strip, expandable to full list (up to 5 shown) when "Details" button is pressed.

    Refresh Rate: Resource ledger refreshes every 1 second via _process().

    HudFactionModule¶

    Located at /home/j/3d-space/scripts/ui/hud_faction_module.gd, manages faction standing notifications and UI updates based on FactionManager signals.

    HudJobsModule¶

    Located at /home/j/3d-space/scripts/ui/hud_jobs_module.gd, manages the discovery journal, cargo manifest modal, jobs board, and discovery notification popups. Provides methods: - toggle_discovery_journal() — Open/close discovery log - toggle_cargo_manifest(ship) — Show cargo details for a ship - toggle_jobs_journal() — Open/close active jobs list - set_discovery_notification(control) — Reference to discovery popup node - set_active_jobs_widget(control) — Reference to active jobs panel

    Input Actions Handled by HUD¶

    The HUD intercepts the following input actions in _unhandled_input(): | Action | Handler | Effect | |--------|---------|--------| | flight_interact | _try_hail_targeted_npc() | Attempt to hail NPC ship | | flight_toggle_discovery_legend | _toggle_discovery_legend() | Show/hide discovery info legend | | flight_journal | _jobs_module.toggle_discovery_journal() | Open/close discovery journal | | flight_cargo_manifest | _jobs_module.toggle_cargo_manifest(_ship) | Toggle cargo manifest modal | | flight_jobs | _jobs_module.toggle_jobs_journal() | Toggle jobs journal |
