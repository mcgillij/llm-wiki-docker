    FPS HUD Instrument Widget System¶

    Summary¶

    The FPS HUD instrument system provides a canvas-layer overlay (FpsHudOverlay) that hosts a collection of UI widgets for the first-person player view. Widgets are created through a factory pattern managed by FpsHudWidgetBootstrap, which handles priority-ordered instantiation, visibility toggling via @export properties on the overlay, and frame-budgeted processing to avoid boot stalls. Individual instrument widgets (lidar, magnetometer, radiation meter, spectrometer, planet science dossier) extend FpsInstrumentWidget and follow a common activate/dismiss lifecycle with mouse capture/release semantics. The system integrates with the radial menu for instrument selection and connects to PlayerAvatarController signals for real-time data display.

    Key Components¶

    FpsHudOverlay — Canvas Layer Host¶

    class_name FpsHudOverlay, extends CanvasLayer. It is the root container for all FPS HUD widgets. Created as a scene instance from FPS_HUD_SCENE in main.gd.

    Container nodes (accessed via @onready): - %ReticleContainer — crosshair reticle - %PromptContainer — interaction prompts, ship interact widget - %DevReadoutContainer — developer debug readout - %CompassRingContainer — directional compass ring - %RadialMenuContainer — radial science menu - %ChatContainer — chat messages - %SpectrometerContainer — spectrometer instrument overlay - %RadiationMeterContainer — radiation meter overlay - %PlanetScienceDossierContainer — planet science dossier overlay

    Export properties control widget visibility: - show_reticle, show_ship_interact, show_interact_prompt, show_dev_readout, show_compass, show_radial_menu, show_spectrometer

    Avatar integration: Connects to PlayerPossession.local_avatar_changed signal. When the avatar changes, it: 1. Disconnects all signals from the old avatar 2. Reconnects to the new PlayerAvatarController: gravity_reading, temperature_reading, surface_reading_progress, surface_reading_completed, radiation_scan_requested, radial_menu_requested, spectrometer_scan_requested, planet_science_dossier_requested, lidar_scan_requested 3. Syncs all existing widgets with the new avatar via _sync_widget_with_avatar()

    Signal handlers: - Gravity/temperature readings → toast notifications via %ToastAnchor/ToastNotification - Surface reading progress/completed → toast + screen flash effect (ColorRect tweened to alpha 0) - Scan requests → resolve and activate corresponding instrument widget - Radial menu request → open FpsRadialMenu

    FpsInstrumentWidget — Base Class¶

    class_name FpsInstrumentWidget, extends Control. All instrument overlays inherit from this base. Key contract:

    Lifecycle: 1. activate(avatar_node) — Sets avatar, adjusts size to viewport, calls _on_activate(), sets visible=true and active=true, enables input processing, notifies Game.notify_popup_opened() 2. dismiss() — Disables input, clears avatar, hides widget, calls _on_dismiss(), notifies Game.notify_popup_closed() and Game.consume_pause_action(), emits dismissed signal

    Key methods: - _unlocks_mouse() — Returns true by default; when true, the HUD captures mouse mode to visible on activate and restores previous mode on dismiss - handle_scan() — Override point for instrument-specific scan logic - _build_backdrop(alpha) — Creates a semi-transparent black backdrop ColorRect (default alpha 0.35) - _build_label(parent, text, font_size, color) — Helper to create centered labels

    Signal: dismissed — Emitted when the widget is dismissed; HUD overlay uses this to track the active instrument and restore mouse mode.

    FpsHudWidgetBootstrap — Factory Orchestrator¶

    class_name FpsHudWidgetBootstrap, extends RefCounted. Manages widget creation with priority ordering and frame-budgeted processing.

    Priority levels: - CRITICAL (0): Reticle, ShipInteract, InteractPrompt - HIGH (1): DevReadout - MEDIUM (2): CompassRing, RadialMenu, Spectrometer, RadiationMeter, PlanetScienceDossier

    Registration: _register_widget_bootstraps() in FpsHudOverlay registers each widget with its factory callable and optional visibility property name. Widgets with empty visibility properties are always created; others respect the corresponding show_* export property.

    Frame budgeting: process_next() processes one widget per frame, stopping if the frame time exceeds _time_budget_ms (5ms). This prevents HUD boot from stalling the game's main loop. The bootstrap is called each frame via FpsHudOverlay._process().

    FpsHudWidgetFactories — Widget Factory Functions¶

    class_name FpsHudWidgetFactories, extends RefCounted. Static factory methods that create and parent widgets to their respective containers:

    Factory Widget Type Container Notes
    factory_reticle ColorRect (crosshair) reticle_container Simple 4x4 white rect at center
    factory_ship_interact FpsShipInteractWidget prompt_container Preloaded script, shows ship re-enter prompts
    factory_dev_readout FpsDevReadoutWidget dev_readout_container Debug readout for development
    factory_interact_prompt FpsInteractPromptWidget prompt_container "Press F — {name}" interaction prompts

    factory_radiation_meter FpsRadiationMeterWidget radiation_meter_container Radiation level display
    factory_planet_science_dossier FpsPlanetScienceDossierWidget dossier_container Comprehensive planet science data

    Instrument Activation Flow¶

    When a scan is requested (e.g., via PlayerAvatarController.spectrometer_scan_requested):

    1. HUD resolves the widget instance (from bootstrap or container children)
    2. Disconnects old result handler, connects new one
    3. Calls widget.configure(profile, deposits) with scan data
    4. Calls _activate_instrument(widget) which:
    5. Dismisses any previously active instrument
    6. Captures mouse mode if _unlocks_mouse() returns true
    7. Calls widget.activate(_local_avatar)

    When the user dismisses an instrument (e.g., presses Escape): 1. Widget emits dismissed signal 2. HUD restores previous mouse mode, clears _active_instrument 3. Widget notifies Game.notify_popup_closed() and Game.consume_pause_action()

    Surface Notification Widgets¶

    Toast notifications (ToastNotification) display transient messages for: - Gravity readings (white text, 8s duration) - Temperature readings (orange tint, 8s duration) - Surface reading progress (amber, 10s duration) - Scan completion (custom color + screen flash effect)

    The screen flash creates a full-screen ColorRect that tweens from the given color to transparent over 0.4s, then queues for freeing.

    Relationships¶

    • Consumed by player-possession-system for avatar lifecycle management
    • Displays data from instruments controlled by PlayerAvatarController (gravity ball, thermal penetrator, gas sniffer, seismic thumper, magnetometer drifter)
    • Radial menu integrates with avatar's perform_science_action(index) method
    • Toast notifications connect to avatar signals: gravity_reading, temperature_reading, surface_reading_progress, surface_reading_completed
