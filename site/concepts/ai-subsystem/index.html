    AI Subsystem — Behavior Trees, Flight Controllers & Steering¶

    Overview¶

    The NPC AI system governs how enemy ships and patrol wings behave in the game world. It combines a behavior tree framework for decision-making with an AI flight controller that extends the player's FlightController to handle movement, combat maneuvers, and formation flying. The system is data-driven via EnemyData profiles that define personality traits (aggression, risk tolerance, accuracy variance).

    Behavior Tree System¶

    Located at /home/j/3d-space/scripts/ai/ai_behavior_tree.gd (151 lines), the BT framework implements a classic recursive behavior tree with three status codes: SUCCESS, FAILURE, and RUNNING.

    Node Types¶

    Node Type Behavior
    Action Leaf Executes a Callable, returns its result
    Condition Leaf Evaluates a predicate, returns SUCCESS/FAILURE
    Sequence Composite Ticks children left-to-right; fails on first FAILURE, runs on first RUNNING, succeeds when all succeed
    Selector Composite Ticks children left-to-right; succeeds on first SUCCESS, runs on first RUNNING, fails when all fail
    All Composite Ticks all children; fails on any FAILURE, returns RUNNING if any child is running, succeeds only when ALL succeed

    Tactical Condition Nodes¶

    Specialized conditions for combat decision-making:

    • BTCheckHullThreshold — Returns SUCCESS if owner's hull < threshold (triggers retreat/evade)
    • BTCheckShieldThreshold — Returns SUCCESS if shields >= threshold (enables aggressive behavior)
    • BTCheckTakingFire — Returns SUCCESS when under attack (triggers evasion/flanking)
    • BTCheckDistanceRange — Returns SUCCESS if target distance within [min, max] range
    • BTCheckAngleBehind — Returns SUCCESS if target is behind the owner (> angle_threshold radians)

    Behavior Trees per State¶

    Each AI state has its own behavior tree: - _patrol_tree — Waypoint navigation, idle behaviors - _pursue_tree — Target acquisition, chase logic - _orbit_strafe_tree — Orbital combat with strafing patterns - _evade_tree — Evasion when under fire or low hull - _flank_tree — Flanking maneuvers to target rear arc - _retreat_tree — Retreat when critically damaged

    AI Flight Controller¶

    Located at /home/j/3d-space/scripts/ai/ai_flight_controller.gd (3,176 lines), this extends FlightController and adds NPC-specific behavior on top of the shared physics model.

    AI States¶

    enum AIState { PATROL, PURSUE, EVADE, FLANK, RETREAT }

    States are managed via an FW_StateMachine with explicit transitions between behaviors. State transitions have a minimum interval (2.0s) to prevent rapid oscillation.

    Patrol Behavior¶

    • Follows waypoints defined in a PatrolPoints node child of the ship
    • If no patrol points exist, creates two default points (PointA/PointB) as fallback
    • Uses patrol_speed multiplier on base speed
    • Can be configured via behavior_type export string

    Pursuit Behavior¶

    Two pursuit modes: 1. Chase — Direct pursuit of target with lead prediction 2. Orbit-strafe — Maintains orbit radius (~600m, ~50% weapon range) while strafing laterally

    Key parameters: - pursue_speed_multiplier (1.15× base speed during chase) - orbit_radius (600m), orbit_tolerance (±100m for smooth transitions) - pursue_lost_timeout (3.5s before considering target lost) - min_separation_distance (18m to avoid collision with other ships)

    Target Selection & Scoring¶

    Targets are scored using a weighted formula in AiCombatTuningHelpers.score_target(): 

    score = (dist_score × dist_weight + angle_score × angle_weight + threat_score × threat_weight + random × random_weight) / total_weight

    Weights are configurable per AI: - target_weight_distance (0.5 default — prefers closer targets) - target_weight_angle (0.25 — prefers targets in front arc) - target_weight_threat (0.2 — prioritizes last attacker) - target_weight_random (0.05 — adds unpredictability)

    Combat Maneuvers¶

    Flanking: Arcs to target's rear arc using flank_min_distance_factor and flank_max_distance_factor of weapon range. Alternates left/right via _flank_arc_direction.

    Evasion: Triggered when taking fire or hull below threshold. Lasts 2 seconds with 3-second cooldown. Tracks previous state to resume after evasion completes.

    Retreat: Activated when hull is critically low. Flies away from target until shields/hull recover.

    Separation & Obstacle Avoidance¶

    • Separation: Each AI maintains a separation_velocity vector that pushes it away from nearby ships (radius: 80m, strength: 16.0)
    • Obstacle avoidance: Ray-casts forward using obstacle_probe_distance (120m), applies lateral and vertical avoidance forces

    Boost Timing & Fire Cadence¶

    Boost behavior: Random intervals between 4–8 seconds, lasting 2 seconds at 1.6× speed multiplier. Scheduled via _schedule_next_boost().

    Fire cadence: Burst-fire pattern with randomized durations: - Burst duration: 0.6–1.4 seconds - Cooldown: 0.4–0.9 seconds - Grace period: 0.2s after solution is lost to allow continued firing

    Personality System (from EnemyData)¶

    Each AI can be assigned an EnemyData profile that sets personality traits in AIContext: - personality_aggression (0.0–1.0) — How aggressively the AI pursues targets - personality_risk_tolerance (0.0–1.0) — Willingness to engage at close range - personality_accuracy_variance (default 0.15) — Random variance in weapon aim

    Wing Integration¶

    AI ships can be members of patrol wings: - Connected to PatrolManager.wing_member_lost signal when assigned a wing_id - _wing_members array caches sibling wing nodes for formation awareness - Wing member loss triggers state re-evaluation

    Steering Behaviors¶

    Located at /home/j/3d-space/scripts/ai/ai_spatial_steering_helpers.gd (52 lines), provides static helper methods for combining multiple steering forces into a single desired velocity.

    Blended Velocity Computation¶

    compute_blended_desired_velocity(
    tactical_dir, tactical_speed,      # Primary movement goal
    lateral,                           # Strafe component
    separation_enabled, separation_velocity,  # Collision avoidance with other ships
    obstacle_avoidance_enabled, obstacle_avoidance_velocity,  # Obstacle avoidance
    separation_max_speed, obstacle_avoidance_max_speed
) -> Vector3

    The blending strategy: 1. Start with tactical direction × speed 2. Add lateral strafe component 3. Add separation force at 60% weight if enabled 4. Blend obstacle avoidance at 65% weight (replaces tactical direction if strong enough) 5. Cap total speed to the maximum of all individual caps

    Separation Velocity¶

    compute_separation_velocity(count, accum, max_speed, strength) -> Vector3
    Accumulates repulsion vectors from nearby ships within separation_radius, normalizes and scales by count × strength (capped at max_speed).

    Obstacle Avoidance Velocity¶

    compute_avoidance_velocity(hit_count, away_accum, max_speed, strength) -> Vector3
    Scales avoidance force linearly with hit count — more hits = stronger evasion response.

    Target Prediction¶

    Located at /home/j/3d-space/scripts/ai/ai_target_velocity_helpers.gd (41 lines), provides duck-typed velocity extraction from target objects:

    enum TargetVelocityMethod { NONE, GET_LINEAR_VELOCITY, GET_VELOCITY, GET_PROPERTY }

    The system tries methods in order of preference: 1. get_linear_velocity() — Preferred for CharacterBody3D targets 2. get_velocity property — Fallback for custom velocity implementations 3. velocity property — Last resort

    This allows the AI to predict target intercept points by extrapolating position using the extracted velocity over pursue_lead_time (0.35s default).

    Relationships¶

    • Extends flight-controller — inherits all physics, boost, and travel mode behavior
    • Uses fw-state-machine for state transitions between PATROL/PURSUE/EVADE/FLANK/RETREAT
    • Consumed by npc-simulation-service for NPC actor lifecycle management
    • Data-driven via faction-data resources (EnemyData profiles define personalities)
    • Integrates with combat-system through weapon firing and targeting manager
