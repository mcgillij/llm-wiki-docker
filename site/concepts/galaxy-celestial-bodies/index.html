    Galaxy & Celestial Bodies — Star Types, Planets, Gates, Phenomena¶

    Overview¶

    This page documents the data-driven resource system that defines every celestial body in a procedurally generated galaxy. The pipeline flows from world-generation → individual galaxy-data configuration → per-system resources for stars, planets, asteroid belts, and jump gates. All resources are Resource subclasses with @export fields, enabling designers to author presets as .tres files while the generator creates instances deterministically from a galaxy seed.

    Star Types & Properties¶

    Stars are defined by StarData (scripts/resources/star_data.gd, 436 lines) and drive lighting, visual appearance, habitable zone calculations, and planet placement logic.

    Spectral Classification (Morgan-Keenan + Exotic)¶

    The system uses the standard OBAFGKM sequence plus six exotic end-state types:

    Class Color Temperature Range Mass (solar) Radius (solar) Luminosity
    O Blue >30,000 K 20.0 M☉ 10.0 R☉ 10,000 L☉
    B Blue-white 10,000–30,000 K 10.0 M☉ 5.0 R☉ 1,000 L☉
    A White 7,500–10,000 K 2.5 M☉ 1.8 R☉ 40 L☉
    F Yellow-white 6,000–7,500 K 1.3 M☉ 1.2 R☉ 4 L☉
    G Yellow (Sun) 5,000–6,000 K 1.0 M☉ 1.0 R☉ 1 L☉
    K Orange 3,500–5,000 K 0.7 M☉ 0.7 R☉ 0.4 L☉
    M Red (dwarf) <3,500 K 0.3 M☉ 0.3 R☉ 0.02 L☉
    White Dwarf White halo 10,000–60,000 K 0.6 M☉ 0.01 R☉ 0.01 L☉
    Neutron Star Needle ~1,000,000 K 1.4 M☉ ~10 km 0.1 L☉
    Black Hole Devouring (dark) 0 K (no light) 10.0 M☉ 0 R☉ 0 L☉
    Dark Star Occult ~500 K 3.0 M☉ 1.6 R☉ 0.02 L☉
    Red Giant Bloated red ~3,000 K — expanded —
    Blue Giant Electric blue ~20,000 K — — —
    Supergiant Cataclysmic 8,000–15,000 K — — —

    Size / Evolutionary Classification¶

    Stars also carry a SizeClass that affects visual scale and light intensity independently of spectral type:

    • Dwarf → base size ×1.0, light intensity 0.5×
    • Main Sequence → base size ×1.0, light intensity 1.0×
    • Subgiant → base size ×1.0, light intensity 1.5×
    • Giant → base size ×2.0, light intensity 2.0× (triggers proximity point light)
    • Supergiant → base size ×3.0, light intensity 3.0×
    • Hypergiant → base size ×5.0, light intensity 4.0×

    Visual Families¶

    Each star maps to a VisualFamily that selects its shader/material preset:

    Family Trigger Description
    Solar Calm G/K/M main sequence Steady yellow/orange glow
    Hot Plasma O/B/A Intense blue-white with turbulence
    Magnetic Active F-type Variable surface activity
    Aged Quiet K/M evolved Dim, stable red-orange
    Bloated Red Red Giant Expanding cool envelope
    Electric Blue Blue Giant Violent ionization glow
    Cataclysmic Supergiant Supergiant Pulsating, multi-layered emission
    White Dwarf Halo White Dwarf Compact bright core with faint halo
    Needle Neutron Neutron Star Ultra-compact, polar beam hints
    Devouring Black Hole Black Hole Accretion disk, gravitational lensing
    Occult Dark Star Dark Star Barely visible, subtle distortion

    Physical Computation¶

    StarData provides several computed properties:

    • Temperature — auto-calculated from spectral class with ±10% randomization seeded by (spectral_class, size_class) hash for determinism.
    • Light color — black-body approximation mapping temperature to RGB (deep red at 1000 K → blue-white above 30,000 K). Visual color is boosted 1.3× saturation for better appearance.
    • Habitable zone — computed as inner = √luminosity × 0.95 AU, outer = √luminosity × 1.37 AU. Used by planet placement to determine which planets can support life.
    • Light intensity — clamp(base_intensity × √luminosity, 0.5, 5.0) where base depends on size class.
    • Visual size — 50 × √radius_solar with giant/supergiant/hypergiant multipliers (2×/3×/5×).

    Creation from Galaxy System Data¶

    Stars are instantiated via StarData.from_star_type(star_type: int) which maps the integer type codes from GalaxySystemData.StarType enum (0–10) to spectral classes with appropriate default mass/radius/luminosity values.

    Planets & Orbital Bodies¶

    Planets are defined by SystemPlanetData (scripts/resources/system_planet_data.gd, 235 lines). Each planet carries orbital parameters, physical properties, and resource deposits.

    Planet Types¶

    Type Description Atmosphere Surface
    Rocky Terrestrial (Earth-like) Standard/Thin/Thick Rocky surface with terrain
    Gas Giant Jupiter-like Thick (H/He) No solid surface — gaseous atmosphere only
    Ice World Frozen outer system Thin/Standard Frozen ice crust, possible subsurface ocean
    Lava World Inner system molten None/Thin Molten surface with volcanic activity
    Ocean World Water-covered Standard/Thick Global liquid water ocean
    Desert World Arid sandy Thin/Standard Sandy desert, minimal water
    Toxic World Poisonous atmosphere Toxic/Corrosive Toxic surface chemistry
    Barren Dead rock None Barren rock, no atmosphere

    Orbital Parameters¶

    Each planet stores:

    • orbit_distance_au — distance from star in AU (used for placement and habitable zone checks)
    • radius_earth / mass_earth — relative to Earth (1.0 = Earth)
    • surface_gravity — derived from mass/radius, affects landing physics
    • axial_tilt — degrees, affects seasonal variation
    • orbital_period_days — year length in Earth days
    • rotation_period_hours — day length

    Atmosphere Types¶

    Five atmosphere categories affect visuals and gameplay: None, Thin (trace), Standard (Earth-like pressure), Thick (high pressure), Toxic (poisonous gases), Corrosive (acidic). Gas giants always report "no solid surface" regardless of atmosphere.

    Resources & Inhabitation¶

    Planets can carry resource_deposits (dictionary mapping resource name to abundance 0–1) and may be marked as inhabited with a population_level (0–10 log scale). The has_significant_resources() method returns true when total deposit value exceeds 0.5.

    Habitable Zone Integration¶

    is_in_habitable_zone(habitable_zone: Vector2) checks whether the planet's AU distance falls within its star's computed habitable zone bounds (from galaxy-data StarData.get_habitable_zone()). This is used by the generator to place life-bearing planets in appropriate orbits.

    Jump Gates¶

    Jump gates are defined by GateData (scripts/resources/gate_data.gd, 94 lines) and enable travel between systems or scenes.

    Destination Types¶

    Type Use Case Target Format
    SCENE_FILE Traditional level transitions Path to .tscn file (e.g., res://scenes/level_b.tscn)
    SYSTEM_COORDINATES Procedural system jumps Vector3i grid coordinates in the galaxy

    Gate Properties¶

    • gate_id / target_gate_id — unique identifiers for bidirectional linking between gates
    • bi_directional — if true, both gates can trigger travel to each other
    • one_way — gate deactivates after a single use (for story-gated progression)
    • requires_key — optional key string that must be held to activate the gate
    • spawn_offset / spawn_rotation — where and how the ship appears on arrival

    Procedural Dimensions¶

    Gates have configurable ring geometry: ring_major_radius (default 500), ring_minor_radius (20), and portal_radius (450). These are used by procedural gate mesh builders to scale the visual model.

    Asteroid Belts¶

    Asteroid belts are defined by AsteroidBeltData (scripts/resources/asteroid_belt_data.gd, 304 lines) which converts at runtime into AsteroidFieldPreset instances for actual generation.

    Composition Types & Associated Minerals¶

    Composition Primary Minerals Visual Appearance
    Rocky Silicon, magnesium, iron Brown/tan, rough (0.55), 8 radial segments
    Metallic Iron, nickel, cobalt, platinum Silver/gray, smooth metallic (0.3 roughness)
    Icy Water ice, ammonia, methane White/blue-white, glossy (0.15 roughness)
    Mixed Iron, silicon, water ice Mixed brown, moderate roughness
    Carbonaceous Carbon, organic compounds, water Dark gray/black, very rough (0.95)

    Belt Geometry¶

    • orbit_distance_au — center of belt ring from star
    • radial_width_au / thickness_au — radial spread and vertical thickness in AU
    • density (0–1) — controls asteroid count (600–2200 range), spacing, and size variation
    • mineral_richness (0–1) — multiplier for resource value of mined asteroids

    Shape Modes¶

    The belt converts to one of two runtime shape modes:

    • DISC — flat ring around the star (standard asteroid belts). Uses Gaussian density falloff from center.
    • CLUSTER — compact spherical cluster (debris fields from recent collisions). Single anchor with spread radius.

    Density Profile & Generation Parameters¶

    At runtime, to_field_preset() converts belt data into an AsteroidFieldPreset with:

    • Density: 0.00004–0.00018 per 1000 m³ (scales with density parameter)
    • Count: 80–320 minimum, 600–2200 maximum asteroids
    • Size range: 0.5× to 3.5× base scale with power-curve distribution skewed toward small objects
    • Large object chance: 3–6% for rare oversized asteroids (up to 14× scale)
    • Mineral weights: composition-specific resource value assignments

    Orbital Mechanics¶

    get_orbital_period_years() applies Kepler's third law: T = √(a³ / M) where a is orbital distance and M is star mass. This gives realistic year lengths for different belt positions.

    Nebulae & Background Phenomena¶

    Nebulae are defined by BackdropNebulaData (scripts/backdrop/backdrop_nebula_data.gd, 7 lines) — a minimal resource that references a pre-built scene and transforms:

    • scene (PackedScene) — the nebula visual to instantiate
    • position / rotation_degrees / scale — placement in world space

    Nebulae are placed as backdrop elements by the level generator, typically at system edges or along sightlines between stars and planets. They use billboard rendering and reactive sky shader parameters (see backdrop-system).

    Galaxy-Level Configuration¶

    The galaxy itself is configured via GalaxyData (scripts/resources/galaxy_data.gd, 244 lines), which controls the large-scale structure that all celestial bodies inhabit:

    • Bounds: Default 10×10×10 grid = 1,000 possible system locations
    • System spacing: 100 light-years between grid points (configurable)
    • Connectivity radius: 150 LY for standard jump drives
    • Warp effects: Twist (65° max), arm-like radial banding (4 arms), noise displacement, and vertical disk flattening — all driven by galaxy_warp_strength (0–1)

    The galaxy seed (galaxy_seed: int = 12345) ensures deterministic generation: the same seed always produces the same star types, planet orbits, belt positions, gate placements, and faction territories. See world-generation for the full pipeline from galaxy seed through system generation to POI placement.

    Key Files Reference¶

    File Lines Purpose
    scripts/resources/star_data.gd 436 Star spectral/size classes, visual families, physical properties, habitable zone
    scripts/resources/system_planet_data.gd 235 Planet types, atmospheres, orbital params, resources, inhabitation
    scripts/resources/gate_data.gd 94 Jump gate destinations, spawn transforms, bidirectional/one-way logic
    scripts/resources/asteroid_belt_data.gd 304 Belt composition, density, mineral weights, runtime preset conversion
    scripts/backdrop/backdrop_nebula_data.gd 7 Nebula scene reference + transform
    scripts/resources/galaxy_data.gd 244 Galaxy bounds, warp effects, connectivity, density curves

    Relationships¶

    • Consumed by world-generation for deterministic system creation from galaxy seed
    • Visuals driven by backdrop-system (planet shaders, nebula billboards) and vfx-visual-effects (star glow, accretion disk effects)
    • Planet resources feed into economy-trade via the market popup system at stations
    • Jump gates enable navigation between systems in galaxy-map-ui
    • Star habitable zone data determines which planets are marked as inhabited in factions-lore
