    Economy & Trade — Commodities, Markets, Jobs System¶

    Overview¶

    The economy system is a data-driven market simulation where players buy and sell commodities at stations, sell discovered exploration data for credits, accept jobs/missions from faction patrons, and manage cargo space. The core engine is TradeService (TS, 1900 lines), an autoload Node that handles all pricing logic, inventory management, reputation effects, and multiplayer RPC channels. Credits are managed by the separate CreditsManager (CM) singleton which wraps PlayerStateManager.

    Commodities¶

    Commodities are defined by CommodityData resources (scripts/resources/commodity_data.gd, 22 lines) — a minimal resource with five export fields:

    Field Type Description
    commodity_id String Unique identifier (e.g., "iron", "water_ice")
    display_name String Human-readable name
    category String Market category for filtering and station preferences
    base_price int Starting price in credits per unit
    volume int Cargo space consumed per unit (SCU)
    legal bool Whether the commodity is legal at all stations
    tags PackedStringArray Additional classification tags

    Default Commodity Catalog¶

    TradeService ships with 21 default commodities loaded from .tres files under res://assets/economy/commodities/:

    Raw Materials: iron, silicon, nickel, platinum, cobalt, copper, gold, magnesium Ice & Volatiles: water_ice, ice, ammonia, methane, carbon, organic_compounds, water Specialty: salvage, supplies, data, nav_data, exotics, limpet_drone

    Each commodity is registered into an internal _commodity_catalog dictionary keyed by commodity_id. The catalog supports category queries via get_all_categories() which returns a sorted list including "all" as the first entry.

    Station Inventory & Market Profiles¶

    Economy Profile Data¶

    Market behavior is tuned by EconomyProfileData (scripts/resources/economy_profile_data.gd, 38 lines) which defines two groups of parameters:

    Inventory Tuning: - Per-station-type base inventory counts (11 types): outpost=24, station=48, mega=96, military=64, research=36, mining=72, agricultural=52, industrial=80, habitat=44, prison=28, scientific=40 - Global multiplier (0.25–3.0×) applied to all inventory counts - Variance range (min/max) for random stock variation per commodity - Price modifier floor (0.6× minimum price) - Minimum stock threshold (default 3 units)

    Market Events: - Demand spike chance (0–100%, default 20%) — increases sell prices, decreases buy prices - Oversupply chance (0–100%, default 20%) — decreases sell prices, increases buy prices - Event modifier ranges for sell and buy sides

    Station Category Profiles¶

    Each station type has a built-in category preference profile that determines which commodity categories are accepted, preferred, discouraged, or restricted:

    Station Type Preferred Discouraged Restricted
    Mining raw, salvage data —
    Industrial raw, trade_goods, consumable artifact —
    Research/Science data, artifact salvage —
    Military consumable, salvage artifact —
    Agricultural trade_goods, consumable artifact —
    Prison consumable artifact artifact
    Outpost consumable, trade_goods artifact —

    These profiles can be overridden per-station via metadata keys: market_allowed_categories, market_preferred_categories, market_discouraged_categories, market_restricted_categories.

    Market Badges¶

    The UI displays color-coded badges on each commodity based on the station's category profile and legality status. Colors are defined as constants in StationMarketPopup:

    Badge Color Meaning
    Allowed default (light blue) Station accepts this category
    Preferred green (0.48, 1.0, 0.56) Better buy/sell rates at this station
    Discouraged orange (1.0, 0.72, 0.34) Weaker rates
    Restricted red (0.95, 0.38, 0.38) Cannot trade this category here
    Black Market orange (1.0, 0.5, 0.32) Contraband accepted at black market stations
    Illegal red Contraband blocked by security
    Low Stock — Station has ≤8 units remaining
    Demand Spike / Oversupply — Active market event label

    Price Computation¶

    Unit prices are computed as a product of seven multiplicative factors applied to the commodity's base_price:

    Sell Price (player sells to station)¶

    sell_unit_price = base_price × market_factor × standing_factor × station_category_factor × faction_bias_factor × faction_policy_factor × event_factor × legality_factor
    Factor Formula Range Effect
    Market clamp(0.72 + security×0.18 - volatility×0.22, 0.45, 1.05) 0.45–1.05 High security → better sell prices; high volatility → worse
    Standing Reputation-based multiplier (see below) 0.75–1.3 Allied = 1.1× (better), Hostile = 0.75× (worse)
    Station Category Preferred = 1.12×, Discouraged = 0.88× 0.88–1.12 Station preference for commodity category
    Faction Bias Per-faction type-specific modifier varies Faction values certain discovery types more highly
    Faction Policy Per-faction buy/sell policy multiplier 0.94–1.12 Federation buys at 0.97×, Pirates buy at 1.12×
    Event Demand spike = 1.18× sell, Oversupply = 0.86× sell varies Temporary market conditions
    Legality Black market stations: +8% for contraband; high security: blocked 0.82–1.34 Contraband handling based on station security

    Buy Price (player buys from station)¶

    buy_unit_price = base_price × market_factor × standing_factor × station_category_factor × faction_bias_factor × faction_policy_factor × event_factor × legality_factor
    Factor Formula Range Effect
    Market clamp(1.05 + volatility×0.35 + (1-security)×0.2, 0.9, 2.0) 0.9–2.0 High volatility → higher buy prices; low security → higher
    Standing Allied = 0.85×, Hostile = 1.3× 0.85–1.3 Allied = cheaper to buy, Hostile = expensive
    Station Category Preferred = 0.9× (cheaper), Discouraged = 1.12× 0.9–1.12 Inverse of sell factor — preferred goods are cheaper to buy

    Standing Modifiers¶

    Based on PlayerReputationClass.Standing enum:

    Standing Sell Modifier Buy Modifier
    Allied 1.10× (better) 0.85× (cheaper)
    Friendly 1.05× 0.93×
    Neutral 1.00× 1.00×
    Unfriendly 0.90× 1.15×
    Hostile 0.75× (worse) 1.30× (expensive)

    Faction Trade Policies¶

    Each faction has a global buy/sell policy that applies to all commodities:

    Faction Buy Modifier Sell Modifier
    Federation 0.97× 1.01×
    Empire 1.04× 0.98×
    Merchants 0.94× (best buyer) 1.04×
    Collective 0.99× 1.05×
    Pirates 1.12× (highest seller price) 1.06×
    Void Church 1.00× 1.03×
    Axiom 1.03× 1.05×

    Illegal Trade & Black Markets¶

    Contraband handling depends on station security level and whether the station has a "Black Market" service:

    • Security ≥ 0.65 without black market: illegal commodities are completely blocked
    • Security < 0.45: slight discount on contraband (0.95× sell, 1.18× buy) — low-security stations tolerate some smuggling
    • Black Market stations: +8% premium for selling contraband, +20% markup for buying
    • Medium security (0.45–0.65): 0.82× sell / 1.34× buy penalties

    Station Services¶

    Beyond commodity trading, TradeService handles station services (refuel, repair, restock) via request_station_service(). Service costs are passed as parameters and deducted from player credits through the CreditsManager. In multiplayer, service requests are routed to the authoritative server via RPC channel 1 (reliable).

    Discovery Value & Data Broker Sales¶

    Players can sell discovered exploration data at stations with a "Data Broker" or "Information" service. This is handled by quote_sell_discovery_bundle() and execute_sell_discovery_bundle().

    Discovery Types & Values¶

    Discoveries are classified into 7 types, each with base values defined in the discovery value profile:

    Type Key Examples
    System "system" Star systems, nebulae
    Planet "planet" Planets, moons
    POI "poi" Points of interest
    Gate "gate" Jump gates
    Asteroid Belt "asteroid_belt" Belt clusters
    Anomaly "anomaly" Strange phenomena
    Station "station" Stations, megastructures

    Scan Levels & Multipliers¶

    Each discovery has a scan level that determines how much data is available for sale:

    Level Key Description
    Unknown "unknown" Detected but unscanned
    Detected "detected" Basic detection data
    Basic "basic" Initial scan results
    Detailed "detailed" Comprehensive analysis
    Surface "surface" Full surface mapping

    The sale price is based on the delta between the current scan level and the last sold scan level — players can sell incremental scan upgrades at progressively higher values. A first-discovery bonus (default 1.15×) applies to never-before-sold discoveries.

    Faction Discovery Broker Bias¶

    Each faction has type-specific value multipliers for discovery sales, reflecting their research priorities:

    Faction System Planet POI Gate Anomaly Asteroid Belt Station

    Empire 1.05× — — 1.08× — — 1.08×
    Merchants 1.08× — — 1.12× — — 1.10×
    Collective — 1.04× — — — 1.16× (belt specialists) 1.05×
    Pirates — — 1.10× 1.05× — — 1.04×
    Void Church — — 1.12× — 1.18× (anomaly experts) — —
    Axiom 1.08× — 1.16× 1.10× 1.22× (highest anomaly value) — —

    Discovery Sale Restrictions¶

    Some discoveries can only be sold at stations controlled by specific factions. If a player attempts to sell at an incompatible station, the sale is blocked with a faction-specific error message listing which stations would accept each discovery entry.

    Jobs System¶

    Jobs (missions/contracts) are displayed through JobBoardPresenter (scripts/ui/job_board_presenter.gd, 374 lines), a static utility class that converts event data into structured job entries for the UI.

    Job Types¶

    Type Source Description
    Convoy Escort Merchant convoy events Protect freighters from pirates along a route
    Pirate Raid Station offers / convoy events Attack enemy convoys (player plays as pirate side)

    Job Entry Structure¶

    Each job entry contains:

    • Identity: id, status (active/available), type, title
    • Factions: issuer_faction_id/name, opposition_faction_id/name, player_side (escort/pirates)
    • Progression: stage (staged/active/completed/failed), progress_ratio (0–1), progress_text
    • Location: origin_label, destination_label
    • Objectives: objective_text (convoy-specific: delivered/remaining/lost freighters, cargo pods recovered, pirate kills)
    • Rewards: reward_credits (total pool), projected_share (player's portion of payout_breakdown)
    • Visuals: accent_color (faction-colored), priority, tracked/trackable flags

    Station Job Offers¶

    Stations can generate job offers including: - Convoy contracts — accept to escort merchant freighters between stations - Pirate raids — accept to attack enemy convoys (available at pirate-controlled or low-security stations)

    Credits Management¶

    Credits are managed by CreditsManager (scripts/systems/credits_manager.gd, 99 lines), an autoload Node that wraps PlayerStateManager:

    Method Description
    get_credits() Returns current credit balance
    set_credits(amount) Sets credits (clamped to ≥0)
    add_credits(amount) Adds credits, emits credits_changed signal
    spend_credits(amount) Deducts credits if sufficient, returns bool
    can_afford(amount) Checks if player has enough credits

    The credits_changed(current, delta) signal is emitted on every change and consumed by the HUD economy module for real-time display updates. In multiplayer, credit operations are routed through PlayerStateManager which handles per-player state synchronization.

    UI Components¶

    Station Market Popup (StationMarketPopup, 650 lines)¶

    A popup window (860×560px) with two panels:

    Sell Panel: - ItemList showing player's cargo items filtered by category - Category filter dropdown (All, raw, trade_goods, consumable, data, artifact, salvage, etc.) - Quantity spinbox (1–stack size) - Quote label showing unit price and total credits - Sell button

    Buy Panel: - ItemList showing station's available commodities with badges - Category filter dropdown - Quantity spinbox (up to 999 or stock limit) - Quote label with pricing - Buy button

    Header: Station name, player credits, cargo volume/capacity Footer: Transaction history (last 14 entries), help text explaining badge meanings and station security level

    HUD Economy Module (hud_economy_module.gd)¶

    Integrates credit display and resource ledger into the main HUD overlay. Refreshes every 1 second with current credits and cargo values.

    Jobs Journal & Active Jobs Widget¶

    The jobs system displays active missions in a dedicated widget with progress bars, faction-colored accents, and reward projections. The JobsJournal tracks completed/failed missions for historical reference.

    Relationships¶

    • Consumed by stations-megastructures for market popup and service costs at stations
    • Feeds into economy-trade via the commodity catalog shared with station inventory generation
    • Discovery sales connect to discovery-exploration journal system
    • Credits managed by CM feed into ship-systems fuel/purchase operations
    • Job board integrates with factions-lore for faction-colored UI and patron/opposition data
    • Market events (demand spike/oversupply) are seeded per-station using hash-based deterministic generation
