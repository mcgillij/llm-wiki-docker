    Ship Systems — Shields, Fuel, Cargo, Mining & Targeting¶

    Overview¶

    Ship systems define the configurable attributes and runtime state of every ship in the game. The system is built around a data-driven architecture where ShipData resources hold all static configuration (stats, parts, weapons), while ShipRuntimeState tracks mutable values (current hull, shields, fuel). Parts are modular components that modify base stats when installed into slot-based loadouts.

    ShipData Resource¶

    Located at /home/j/3d-space/scripts/ship_data.gd (814 lines), ShipData is the central configuration resource for all ships. It extends Godot's Resource class and is assigned to every ship node at runtime via apply_ship_data().

    Core Stats¶

    Property Type Default Description
    mass float 1000.0 Base ship mass (affects inertia)
    base_speed float 40.0 Base movement speed
    boost_multiplier float 2.0 Speed multiplier during boost
    rotation_speed Vector3 (1.6, 1.4, 2.0) Yaw/Pitch/Roll rotation rates
    throttle_change_rate float 1.5 Throttle ramp speed
    damping float 2.5 Velocity damping factor
    brake_strength float 3.0 Deceleration rate

    Mass Composition¶

    Total mass is the sum of all component masses: 

    total_mass = base_mass + hull_mass + engine_mass + shield_mass

    Parts System¶

    Ship parts are modular components that modify ship stats when installed into designated slots. Each part extends ShipPart and is applied via typed methods on ShipData.

    Part Types & Slots¶

    Slot ID Type Constant Part Class Purpose
    "hull" — HullPart Hull integrity, mass
    "engine" — EnginePart Speed, boost, jump drive, travel drive
    "shield" SLOT_TYPE_INTERNAL (1) ShieldPart Shield capacity, recharge, visual properties
    "cargo" SLOT_TYPE_INTERNAL (1) CargoBayPart Cargo capacity
    "limpet_bay" — — Limpet/drone storage count
    "hardpoint_N" SLOT_TYPE_HARDPOINT (0) WeaponData Weapon hardpoints (N = 0–7)
    Countermeasure slots SLOT_TYPE_COUNTERMEASURE (3) CountermeasureData Countermeasure groups (default 4)

    EnginePart¶

    Located at /home/j/3d-space/scripts/resources/engine_part.gd (28 lines), defines engine performance and jump drive capabilities.

    Performance multipliers — Applied to base stats: - speed_multiplier — Final speed = base_speed × speed_multiplier × travel_speed_multiplier - boost_multiplier — Boost speed multiplier - throttle_multiplier — Throttle change rate modifier - rotation_multiplier — Rotation speed modifier

    Boost system: - boost_value — Total boost time in seconds (default 5.0) - boost_recharge — Recharge rate in units/second (default 1.0)

    Jump Drive: - has_jump_drive — Whether the engine includes a jump drive - jump_range_ly — Maximum jump distance in light-years (default 20.0) - jump_fuel_efficiency — Fuel consumed per light-year (lower is better, default 1.0) - jump_warmup_time — Preparation time before jumping (default 5.0s) - jump_cooldown_time — Cooldown between jumps (default 10.0s) - fuel_tank_capacity — Fuel tank capacity in units (default 100.0)

    Travel Drive: - travel_speed_multiplier — Multiplier for travel mode speed (default 40.0, so final = base × 40) - travel_spinup_time — Time to reach full travel speed (default 2.0s) - travel_fuel_consumption — Fuel consumed per second in travel mode (default 0.25/s)

    ShieldPart¶

    Located at /home/j/3d-space/scripts/resources/shield_part.gd (16 lines), defines shield properties.

    Property Default Description
    capacity 120.0 Maximum shield hit points
    recharge_rate 18.0 Shield recovery rate per second
    recharge_delay 1.0 Delay before recharge begins after damage
    radius 3.5 Shield bubble radius in world units
    base_alpha 0.08 Visual opacity of shield bubble
    shield_color (0.2, 0.6, 1.0) Shield bubble color
    impact_strength 0.8 Visual impact intensity
    impact_radius 0.6 Impact spark radius
    impact_fade 1.2 Impact visual fade rate

    CargoBayPart¶

    Located at /home/j/3d-space/scripts/resources/cargo_bay_part.gd, defines cargo capacity for a ship's cargo bay module.

    Runtime State: ShipRuntimeState¶

    Located at /home/j/3d-space/scripts/resources/ship_runtime_state.gd (31 lines), this resource tracks mutable values that change during gameplay. It is initialized from ShipData and updated by the ship controller during simulation.

    Fields¶

    Field Type Description
    hull_current / hull_max float Current/max hull integrity
    shield_current / shield_max float Current/max shield capacity
    boost_level / boost_max float Current/max boost energy
    fuel_current / fuel_max float Current/max jump fuel
    cargo_amount int Current cargo units carried
    limpet_count int Available limpets/drones
    destroyed bool Whether the ship is destroyed

    Initialization¶

    func initialize_from_ship_data(ship_data: ShipData) -> void
    Sets all max values from ShipData and initializes current values to maximum. Called when a ship is spawned or loaded.

    Loadout System¶

    The loadout system manages equipment slots and installed parts. It's accessed via ShipData.get_or_build_loadout() which returns a ShipLoadoutData resource.

    Slot Management¶

    func _set_loadout_slot(slot_id: String, part: ShipPart) -> void
    Assigns a part to a named slot in the loadout. Called automatically when parts are applied via apply_hull_part(), apply_engine_part(), etc.

    Module Retrieval¶

    func get_hull_module() -> HullPart
func get_engine_module() -> EnginePart
func get_shield_module() -> ShieldPart
    Retrieves installed parts from the loadout by slot ID. Returns null if no part is installed or loadout is not seeded.

    Validation¶

    func validate_loadout() -> Array[String]
    Returns an array of validation issue strings. Called during ship initialization to catch configuration errors.

    Weapon Groups¶

    ShipData supports up to 8 weapon groups (WEAPON_GROUP_COUNT = 8), each containing one or more weapons with a fire policy.

    WeaponGroupData¶

    Each group has: - weapons: Array[WeaponData] — Weapons in this group - fire_policy: FirePolicy — How weapons fire (SIMULTANEOUS or SEQUENTIAL)

    Fire Policies¶

    Policy Behavior
    SIMULTANEOUS All weapons fire at once
    SEQUENTIAL Weapons fire one at a time, cycling through the group

    Weapon Selection¶

    func set_active_weapon_group(index: int) -> void  # Switch to group N
func get_next_non_empty_group_index(current: int, direction: int) -> int  # Cycle forward/backward
func get_active_weapon_data() -> WeaponData  # Get current weapon

    Countermeasures¶

    Countermeasures are defensive items deployed from countermeasure slots. Each slot holds a CountermeasureData resource.

    Management¶

    func get_countermeasure_groups() -> Array[CountermeasureData]
func get_countermeasure_group(index: int) -> CountermeasureData
func set_countermeasure_group(index: int, countermeasure: CountermeasureData) -> void

    Jump Drive System¶

    Jump drives enable fast travel between systems. The system is integrated into EnginePart and managed through ShipData.

    Key Methods¶

    Method Description
    has_jump_drive() Checks if engine has jump drive
    get_jump_fuel_capacity() Returns fuel tank capacity (from engine or default)
    get_jump_range() Returns max jump distance in light-years
    get_jump_fuel_efficiency() Returns fuel cost per light-year
    get_jump_warmup_time() Returns preparation time before jumping

    Fuel Consumption¶

    Jump fuel is consumed based on: 

    fuel_cost = distance_ly × jump_fuel_efficiency
    If remaining fuel < fuel_cost, the jump cannot be initiated.

    Ship Stats Snapshot¶

    ShipStatsSnapshot (referenced via SHIP_STATS_SNAPSHOT_SCRIPT) provides a computed summary of all ship stats for UI display and AI decision-making: - Total mass breakdown (base, hull, engine, shield) - Speed calculations (base, boosted, travel mode max) - Rotation speeds per axis - Shield capacity and recharge rate - Weapon group summaries

    scripts/resources/shield_part.gd Shield capacity, recharge, visual properties 16
    scripts/resources/cargo_bay_part.gd Cargo bay capacity configuration —
    scripts/resources/hull_part.gd Hull integrity and mass configuration —

    Relationships¶

    • Consumed by flight-controller for physics parameters (mass, speed, rotation rates)
    • Used by combat-system for shield/hull damage application and weapon range
    • Integrated with networking-multiplayer via ShipRuntimeState snapshot fields
    • Drives ui-subsystems HUD displays (shield bar, fuel gauge, cargo indicator)
