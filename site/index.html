    Wiki Index — 3dSpace Technical Knowledge Base¶

    Content catalog. Every wiki page listed under its type with a one-line summary. Last updated: 2026-05-25 | Total pages: 84

    Entities¶

    Core Game Managers¶

    • Galaxy Generator — Generates star systems, planets, and celestial bodies for the galaxy map.
    • Galaxy Navigator (GN) — Zone-based navigation with route shard indexing for fast pathfinding across sectors.
    • Gate Manager — Manages jump gate activation, cooldowns, and destination routing.
    • Jump Manager (JM) — Handles jump travel state machine and transition effects.
    • Level Manager — Loads/unloads levels, manages scene transitions between systems and stations.
    • Spawn Manager — Spawns NPCs, ships, and world objects with lifecycle management.

    Framework Services¶

    • FW Config Service — INI config persistence with typed getters and defaults fallback chain.
    • FW Scene Router — Scene transitions with 10 shader transition presets (wipes, fades, rotation).
    • FW State Machine — Explicit state machine with on_enter/on_exit/on_update/on_event lifecycle.
    • FW Trackable Registry — Registry for tracking and managing FW framework entities.

    Multiplayer Replication¶

    • Flight Controller (ShipController) — Player ship movement, physics model (SharedFlightModel), boost/travel/highway systems, client prediction, multiplayer sync.
    • Remote Ship Controller — Client-side visual proxy for remote ships with SnapshotBuffer interpolation.
    • Shared Flight Model — Deterministic ship physics shared between server and clients.
    • Snapshot Buffer — Interpolation buffer: quaternion slerp + linear lerp for smooth ship state rendering.
    • Ship Visual Proxy — Client-side visual representation of remote ships with VFX integration.
    • Ship Replication Service (SRS) — Server-authoritative ship state replication to clients at 30Hz.
    • Avatar Replication Service (AVRS) — Avatar state replication for player character on foot/inside stations.
    • Multiplayer Session (MPS) — Manages multiplayer session lifecycle, connection handling, and matchmaking.

    World State & Services¶

    • Server World State Service (SWS) — Authoritative server-side world state management for all entities.
    • Ship Config Service (SCS) — Ship configuration persistence: loadouts, parts, and customization data.
    • Player Identity Manager (PIM) — Manages player identity across sessions with faction reputation and progression.
    • Player State Manager (PSM) — Tracks mutable player state: credits, cargo, discovery progress, inventory.

    Economy & Discovery¶

    • Credits Manager (CM) — Credits singleton: balance tracking, transactions, and economy integration.
    • Resource Ledger Manager (RLM) — Resource ledger: tracks commodity holdings, market prices, and trade history.
    • Trade Service (TS) — Trade engine with 21 commodities, 7-factor price computation, station markets, and jobs system.
    • Discovery Manager (DM) — Discovery bubble system (75 LY auto-scan, 500 LY pulse), 7 discovery types × 5 scan levels, probe management.

    Galaxy Management¶

    • Galaxy Save Manager (GSM) — Sectorized save/load v2 for galaxy state with async worker thread and schema migration.
    • Galaxy System Registry (GSR) — Registry of all generated systems with lookup, caching, and streaming support.

    NPC Systems¶

    • Faction Manager — Faction reputation/standing system: tracks player relations across factions.
    • Patrol Manager — NPC patrol wing management: Vee formation, patrol routes, and wing coordination.
    • NPC Simulation Service — Lightweight vs full runtime NPC lifecycle simulation tiers.
    • NPC Promotion Service — Promotes/demotes NPCs based on proximity, hails, and combat performance.
    • NPC Replication Orchestrator — Server-authoritative NPC state replication to clients.
    • NPC World Orchestrator — Coordinates NPC spawning, patrol assignment, and world integration.

    World Runtime & Generation¶

    • World Runtime Manager — Manages runtime world state: active systems, streaming zones, and cleanup.
    • Procedural Level Builder — Procedural generation of stations, megastructures, surface features from recipe data.
    • Star Generator — Generates stars with spectral types (OBAFGKM + exotic), visual families, and habitable zones.
    • Station Docking Manager — Station scan/prompt/menu flow: refuel, repair, restock, service costs.
    • System Event Runtime — Runtime event system for in-game triggers, notifications, and state changes.

    Network & Replication Services¶

    • Network Ship Manager — Server-side snapshot broadcast at 30Hz with distance-based delivery policy (near/medium/far tick divisors).
    • Network Snapshot Generator — Compiles ship state data for network replication: position, velocity, weapon states, packed binary format.
    • Prediction Service — Client-side prediction and interpolation for smooth ship movement between server snapshots.
    • Player Possession Service — Manages transitions between ship control (flight mode) and avatar (on-foot/inside stations).

    Galaxy & Map Services¶

    • Galaxy Cache Service — Caching layer for galaxy map data: sector summaries, route information, hierarchy queries.
    • Local System Map Manager — Manages the local system map UI and navigation within a single star system.

    Combat & Targeting¶

    • Targeting Manager — Combat targeting: target groups, cycling through available targets, lightweight NPC handling.
    • Combat Targeting Solver — Intercept point quadratic solver for combat targeting calculations.

    Chat & Communication¶

    • Chat Service — Networked chat system for multiplayer communication between players and NPCs.

    Framework UI Services¶

    • FW Input Rebind Service — Runtime input action rebinding: keyboard/mouse and controller customization.
    • FW Platform Detection — Detects runtime platform (Steam, standalone, web) and enables platform-specific features.
    • FW Resource Cache — Preload queue and resource caching system for efficient asset loading with LRU eviction.
    • FW Debug Utilities — Structured debug logging with severity levels, category tagging, and optional profiling hooks.
    • FW Context Menu — Reusable context menu component: right-click menus with dynamic item generation.
    • FW Modal Overlay — Modal dialog system: fullscreen dialogs with focus trapping and backdrop dimming.

    Concepts¶

    Architecture & Systems¶

    • Project Architecture Overview — Project overview, directory structure, patterns, and conventions.
    • AI Subsystem — NPC AI: behavior tree framework (Action/Condition/Sequence/Selector), 5 AI states, steering helpers, personality system.
    • Combat System — ShipWeapons controller, weapon runtime classes, TargetingManager, CombatTargetingSolver, heat/ammo systems.
    • Networking & Multiplayer — Authoritative server model, snapshot broadcast at 30Hz, distance-based delivery policy, dynamic delay adaptation.
    • Ship Systems — ShipData resource (814 lines), parts system (engines/shields/cargo/hull), loadout management, weapon groups, countermeasures.
    • FW Framework Addon — ConfigService, SceneRouter, StateMachine, AudioService, NetService, SaveService.
    • FW Framework Additions — New FW_ services: input rebinding, platform detection, resource cache, UI overlays.
    • UI & HUD Subsystems — HudOverlay CanvasLayer (1256+ lines), priority bootstrap system, 28 widget factories, service modules.
    • VFX & Visual Effects System — Thruster sockets, ShipVFX Manager, bloom effects, particle systems.

    World & Content¶

    • World Generation Pipeline — Galaxy → Systems → Objects generation pipeline with sector streaming and zone graph routing.
    • Galaxy Sector Streaming — Sectorized data streaming + caching for efficient galaxy navigation.
    • Surface Features Generation — Procedural cities, landmarks, and outposts on planet surfaces.
    • Backdrop System — Planets, nebulae, landable surfaces, and sky rendering backdrops.
    • Sky Parameter System — Dynamic sky rendering parameters: lighting, atmosphere, weather effects.
    • Factions & Lore — Faction identities, governments, relations, and lore backgrounds.
    • Ships, Parts & Lore — Ship hulls, engines, shields, hardpoints, loadouts, and faction ship rosters.
    • Weapons & Countermeasures — Weapon types (laser/rocket/plasma/mining/probe/limpet), mechanics, and countermeasure systems.
    • NPCs & Patrols — NPC lifecycle simulation, patrol wings (Vee formation), six ship roles with faction job mixes.
    • Stations & Megastructures — StationData resource (11 types, 9 services), 13 megastructure kinds, procedural generation from recipe data.
    • Galaxy & Celestial Bodies — Star types (OBAFGKM + exotic), planet types with atmospheres, jump gates, asteroid belts, nebulae.

    Economy & Exploration¶

    • Economy & Trade — TradeService engine: 21 commodities, 7-factor price computation, station markets, discovery value selling, jobs system.
    • Mining & Limpet Systems — Asteroid mining mechanics, limpet bay management, resource extraction workflows.
    • Discovery & Exploration — Discovery bubble system (75 LY auto-scan, 500 LY pulse), 7 discovery types × 5 scan levels, probe management, sectorized save/load v2.

    Multiplayer & Replication¶

    • Galaxy Navigation Routing — Zone graph + route shard indexing for fast pathfinding across galaxy sectors.
    • NPC Replication & World Orchestration — Server-authoritative NPC replication: state sync, spawning, and world integration.
    • Event Replication System — Server-to-client event broadcasting for game events, notifications, and state changes.

    Player Systems¶

    • Player Possession System — Ship ↔ avatar transitions: player control switching between flight and on-foot modes.
    • Ship Systems Coordinator — Unified ship state management: coordinates subsystems (shields, fuel, weapons, engines).

    Local Simulation¶

    • Local System Traffic — Intra-system NPC traffic simulation: patrol routes, civilian movement, and congestion modeling.
